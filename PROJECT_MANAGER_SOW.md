## 1. PROJECT OVERVIEW

### 1.1 Purpose
The Project Manager will oversee all email marketing operations for multiple clients using the Retention Harbor Analytics & Operations Dashboard. This role is responsible for managing campaign workflows, client communications, approval processes, and ensuring timely delivery of all email marketing initiatives.

### 1.2 System Overview
The Operations Dashboard is a comprehensive platform for managing:
- Email campaigns (one-time sends)
- Automated email flows (welcome series, abandoned cart, etc.)
- SMS campaigns
- A/B tests
- Client approvals and revisions
- Content strategy and creative production

**Platform URL:** `https://analytics.retentionharbor.com/agency/retention-harbor/ops`

## 5. WORKFLOW PROCESSES

### 5.1 New Campaign Workflow

```
1. STRATEGY PHASE (PM)
   ├─ Create campaign in Ops Dashboard
   ├─ Set campaign type, send date, audience
   ├─ Add to monthly calendar
   ├─ Assign to copywriter
   └─ Status: "Content Strategy" → "Copy"

2. COPY PHASE (Copywriter)
   ├─ Write campaign copy in Google Docs
   ├─ Link copy doc to campaign
   ├─ Update status to "Copy QA"
   └─ Notify PM when complete

3. COPY QA PHASE (PM or Copy Lead)
   ├─ Review copy for brand voice, accuracy
   ├─ Request revisions OR approve
   └─ Status: "Copy QA" → "Design"

4. DESIGN PHASE (Designer)
   ├─ Design email in Figma/Canva
   ├─ Upload preview image to campaign
   ├─ Update status to "Design QA"
   └─ Notify PM when complete

5. DESIGN QA PHASE (PM)
   ├─ Review design for brand alignment
   ├─ Check mobile responsiveness
   ├─ Request revisions OR approve
   └─ Status: "Design QA" → "QA"

6. FINAL QA PHASE (QA Team or PM)
   ├─ Final review of copy + design together
   ├─ Check all links, CTAs, personalization
   ├─ Ensure no errors or typos
   └─ Status: "QA" → "Client Approval"

7. CLIENT APPROVAL PHASE (PM)
   ├─ Campaign appears in Client Portal
   ├─ PM notifies client via email/Slack
   ├─ Client reviews and approves/requests changes
   ├─ PM processes feedback
   └─ Status: "Client Approval" → "Approved" OR "Client Revisions"

8. IMPLEMENTATION PHASE (Implementor)
   ├─ Build campaign in Klaviyo
   ├─ Set up segments, schedules, tracking
   ├─ Complete Implementation QA
   └─ Status: "Approved" → "Scheduled"

9. SEND & CLOSE (Automatic)
   ├─ Campaign sends at scheduled time
   ├─ Status: "Scheduled" → "Sent"
   └─ Campaign archived
```

**Total Time:** 5-7 days (strategy to sent)  
**PM Touchpoints:** 5 times (create, copy QA, design QA, final QA, client approval)

---

### 5.2 Client Approval Process

```
1. PM SUBMITS FOR APPROVAL
   ├─ Set campaign status to "Client Approval"
   ├─ Campaign auto-appears in Client Portal
   ├─ Send notification email to client
   └─ Add to approval tracking list

2. CLIENT REVIEWS (24-48 hour window)
   ├─ Client opens Portal
   ├─ Views campaign preview image
   ├─ Reviews copy document
   └─ Provides feedback

3. CLIENT DECISION
   ├─ APPROVE → Status: "Approved"
   │   └─ PM assigns to implementor
   │
   └─ REQUEST CHANGES → Status: "Client Revisions"
       ├─ Client adds revision notes
       ├─ PM reviews feedback
       ├─ PM assigns to copy/design for revisions
       └─ Repeat approval process

4. PM FOLLOW-UP
   ├─ If no response in 24 hours → Send reminder
   ├─ If no response in 48 hours → Schedule call
   └─ If no response in 72 hours → Escalate to account manager
```
